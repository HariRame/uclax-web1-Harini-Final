Q1. Explain the difference between HTML, CSS and JS.

HTML

HTML is the abbreviation of Hyper Text MarkUp Language. The browser orders or places the content as defined in the html page. The content can be text, images, list, links,icons,headings, paragraphs. The root element is the HTML tag. It contains the HEAD and the Body tags where the page title/heading and the content are placed respectively. Each element has attributes like Id, class and name that provides information about the element.

CSS

CSS is the abbreviation for Cascading Style Sheets. It controls how the content will be displayed in the browser ie,. the font size, color, thickness of the border and padding etc. The css can be defined in-line under the STYLE tag in the HTML document or defined in the .css file and then imported in the HTML file. It is good practice to maintain basic page format across the website and the .css files helps acheive that.

<style>
div { color:blue; }
</style>

The above style will be applied to all the DIV tags in the HTML page.

Javascript

Javascript is the language that is used to make the site dynamic by adding functionalities to the html elements or by changing the css of an element or changing the element itself. It can also manipulate the data provided by user (for validation before passing to the server ) or data provided by the server(To display relevant information in the browser).

Q2. What are Single Page Apps (SPAs)? How do they differ from traditional multi-page Web Sites?

Single Page Application

Single Page Applications are websites in which client need not reload the entire page when there is a change in data (may or may not be due to uer action), only that particular resource is refreshed or reloaded.
Example, In the Facebook application, when the user on the Home page, scrolls through it and then clicks the 'Like' button.
Here
  • The user has changed data by clicking the button,
  • hence data is updated in the DB through the API's
  • then it should be seen on the page that the post is Liked.
    • In a single page application, All of the above steps occurs seemlessly without page reload. All the HTML, CSS and Javascript files of the SPA are downloaded by the browser initially, the DOM is created and then saved in the memory. This improves the speed and performance of the application.

    Multi-page Application

    Multi Page Applications have multiple web pages that will be retrieved from the server upon request from the browser.
    Example, In a E -Commerce websiteWhen a user clicks on the image of a product.
    Here
  • The browser requests the server to provide the page about the product.
  • The Server renders a new page to the client which may have new set of HTML,CSS & Javascript files
  • THe browser loads the file and creates the DOM.

    • The main difference between the SPA and the MPA is the in the way the web pages are rendered. SPA design is applied when the application needs to be fast and requires one page. MPA design is applied when the application is complex hence needs multiple pages.
