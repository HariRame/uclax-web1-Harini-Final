Q1. Explain the difference between HTML, CSS and JS.

HTML

HTML is the abbreviation of Hyper Text MarkUp Language. The browser orders or places the content as defined in the html page. The content can be text, images, list, links,icons,headings, paragraphs. The root element is the HTML tag. It contains the HEAD and the Body tags where the page title/heading and the content are placed respectively. Each element has attributes like Id, class and name that provides information about the element.

CSS

CSS is the abbreviation for Cascading Style Sheets. It controls how the content will be displayed in the browser ie,. the font size, color, thickness of the border and padding etc. The css can be defined in-line under the STYLE tag in the HTML document or defined in the .css file and then imported in the HTML file. It is good practice to maintain basic page format across the website and the .css files helps acheive that.

<style>
div { color:blue; }
</style>

The above style will be applied to all the DIV tags in the HTML page.

Javascript

Javascript is the language that is used to make the site dynamic by adding functionalities to the html elements or by changing the css of an element or changing the element itself. It can also manipulate the data provided by user (for validation before passing to the server ) or data provided by the server(To display relevant information in the browser).

Q2. What are Single Page Apps (SPAs)? How do they differ from traditional multi-page Web Sites?

Single Page Application

Single Page Applications are websites in which client need not reload the entire page when there is a change in data (may or may not be due to uer action), only that particular resource is refreshed or reloaded.
Example, In the Facebook application, when the user on the Home page, scrolls through it and then clicks the 'Like' button.
Here
  • The user has changed data by clicking the button,
  • hence data is updated in the DB through the API's
  • then it should be seen on the page that the post is Liked.
    • In a single page application, All of the above steps occurs seemlessly without page reload. All the HTML, CSS and Javascript files of the SPA are downloaded by the browser initially, the DOM is created and then saved in the memory. This improves the speed and performance of the application.

    Multi-page Application

    Multi Page Applications have multiple web pages that will be retrieved from the server upon request from the browser.
    Example, In a E -Commerce websiteWhen a user clicks on the image of a product.
    Here
  • The browser requests the server to provide the page about the product.
  • The Server renders a new page to the client which may have new set of HTML,CSS & Javascript files
  • THe browser loads the file and creates the DOM.

    • The main difference between the SPA and the MPA is the in the way the web pages are rendered. SPA design is applied when the application needs to be fast and requires one page. MPA design is applied when the application is complex hence needs multiple pages.

    Q3. What is the difference between Web Designer, Front End Developer and Back End Developer?

    Web Designer

    Web Designer are professionals who designs the look and the feel of a web application. The usability & accessibility of the features, the layout of the web page, the color, font and the images are created by the Web Designer as Wireframes.

    Front End Developer

    Front end developers are IT Professionals who develop the User Interface of the application as per the Wire frames created by the Web Designer. They use front end programming languages like HTML,CSS, javascript and Packages like React or Angular to develop the user interface of the web application.

    Backend Developer

    Backend developers are IT professionals who develop web services / API's that are required to manipulate or update information between the client and the server. Example, when the user enters the information on a web page and saves, the web service is called to establish connections with a DB, to update the recent information. The back end developer codes this part of the application.

    Q5. What it means for a company to be Agile. What is the purpose of using Agile, and its Pros and Cons.

    Agile is a software development process in which work is planned for a sprint(usually 2 weeks), focusses on continuous development and feedback. It involves collaboration within teams by sprint planning session, daily stand-ups and retrospective meetings.

    The enitre feature is broken down in to small user stories. These user stories are ordered, developed, tested and finally accepted by the product owner.

    Pros of Agile

    With immediate feedbacks, changes in requirements are effectively handled by creating user stories and adding it in backlog. They are then prioritized during the backlog and planning session before they are worked on. This helps in developing applications faster and improves customer satisfaction. It eliminates redundant documentation and delivers working software in short time.

    Cons of Agile

    It is very difficult to plan and allocate resources required as the application may get complex.

    Q6. Define the following roles: Project Manager, Business Analyst, Scrum Master, UX designer, Web Developer/Engineer, Quality Assurance Tester, and DevOps.

    Project Manager

    They manage the resources and risk assessment in the project. They are also the first point of contact for the business leadership.

    Business Analyst

    They design wire frames that explain the feature to be developed. They need to have good understanding of the existing system and how the new enhancements will improve it.

    Scrum Master

    Scrum master helps the team in fascilitaing meetings and conflict managemnt. They help in removing the blockers faced by the team.

    UX designer

    They design prototypes and designs of the application. They make sure the product is easily accessible/usable and visually appealing by the end users.

    Web Developer/Engineer

    They are the one who design, implement/code the application. They can DB engineers, backend, front end or full stack engineers. It also includes Architects who design the systems that are to be developed.

    Quality Assurance Tester

    They test the application for bugs. Raise defects and retest when they the defects are fixed. They execute test manually or create and execute automated scripts.

    DevOps

    They maintain applications. They are also responsible for deploying code from the staging to the production environment.

    GIF

    This bitmap format supports up to 8 bits per pixel which limits its color palette to 256. Not good for photographs an are mostly known these days for simply animated memes.

    PNG

    PNG was designed for use on the Internet and is the most largely used image compression format on the web. PNGs are not for printing professional graphic images.

    SVG

    SVGs are vector graphics that support animation and their behaviors are defined in XML files for they can be searched, indexed, scripted and compressed. Vector graphics are great for logos and line art because they aren't made from pixels and can be infinitely resized without losing quality or increasing in size. SVGs are your best option when creating icons and line art for high-resolution displays.

    Q8. What are testing environments? What is the difference between Local, Dev, QA and Production?

    Test environement mimics the software and hardware of the real time application. It can include test test, databaseweb servers and OS. The development team uses this environment to test the application before relaese to production. The test environment can be categorized as follows.

    Local

    It is the developer's computer where the code exists. After making the necessary changes the developer performs unit tests on a section of code to check its corectness.

    Dev

    After the unit test are done, the changes are pushed in to the Dev environment. It verifies the integration among different sysrems works properly. It doesnt completely mimic the prodiction environment.

    QA

    QA environment is the environemnt where majority of the testing are performed. Smoke tests, integration and regression tests are performed through manually or automated test cases. Most companies have a separate staging environment which is the most stable and closer to the production environment.

    Production

    The production environment is the actual real time application exists. Usually, business leaders create special accounts to test in real time environment to get a sense of user experience.
